# かくきんシステム ～Probablistic Evenizing System～

```
The least common multiple with equal probability of winning, as determined by the cumulative binomial distribution.  
累積二項分布によって決定される、当選確率が等しい最小公倍数。
```

# 将棋というゲームが向き合わなければならない将来必ずやってくる問題

２人で遊ぶ将棋では、Ａさんが強いのか、Ｂさんが強いのかという力比べが行われます。ルールを覚えたばかりのプレイヤー同士が遊ぶと、先手を選んでも後手を選んでも、どっちを選んでも勝率は４９～５１％の差だけなので、先手後手の差ではなく、ＡさんとＢさんの力の差で勝ち負けが決まるという、奇跡のゲームバランスを持っている時期が４００年以上続いています。  

しかし、コンピュータ将棋が発展し、百手先近くまでもの勝ちやすいパターンの調査が進んだことにより、大会の決勝に出てくる強さのコンピュータ同士の対局では、先手を選んだ方が７割近く勝つというように偏ってきました。  
これでは、Ａさんのソフトが強いのか、Ｂさんのソフトが強いのかという力比べではなく、 **くじ引きで先手を引いた方が勝つくじ引きゲーム** になってしまいます。これでは将棋の勝負ではありません。（**将棋の先手勝率７割問題**）  

そこで、どうにかして先手勝率が４９～５１％だった頃に戻したいという強い要請が大会運営者、決勝に参加する競技者から出てきていて、コンピュータ将棋という競技の存続に関わっているとともに、いずれ人間の将棋でも議論される問題と思います。  


# 解決案

ここでは、その解決方法になるであろう方法を示します。  

この方法は、勝つ確率を均等に戻すシステムなので、**かくきんシステム（確均システム；Probablistic Evenizing System）**とでも仮に名付けて呼んでおくことにします。  

やり方は、先手の１勝と後手の１勝で異なる勝ち点を設定し、そして両者には共通する目標となる勝ち点数を設定し、その目標の点数を先に達成した方を勝ちにする方法です。  

例：　先手勝率の偏りが７０％の場合なら、先手での勝ち１つを１点、後手での勝ち１つを２点とし、２点先取したプレイヤーの勝ちとします。１局毎に先後は入れ替えるものとします。対局は最短で１局、最長で３局で決着します。（※千日手や、手数上限による引き分けをどう扱うかは今は考慮していません）  

将棋のルールに手を加える必要がないことと、  
大会運営側の勝ち星の管理方法を変えるだけでよいのがメリットです。  

||従来のｎ本勝負|かくきんシステム|
|----|----|----|
|対局数|奇数|奇数も偶数もある|
|手番|１局ごとに先後を交代（先後交互制）|１局ごとに先後を交代（先後交互制）してもいいし、先手はずっと先手、後手はずっと後手（先後固定制）でも使えます（**※注１**）|
|勝つ必要がある局数|過半数（**注２**）|どちらかのプレイヤーが目標の勝ち点に達するまで行う|
|最長の対局数|ｎ|後手番があと１勝のときに、先手番の勝ちが決まったときの対局数（先後固定制）。<br>ＡさんとＢさんの両者が先手で勝ち続けた対局数（先後交互制）|
|最短の対局数|ｎ／２（小数点以下切り上げ）|後手番が全勝したときの対局数（先後固定制）。<br>後手で始まるＢさんだけが勝ったときの先手番と後手番を足した対局数（先後交互制）|
|複数回やる意義|一発勝負では測り損ねるような、まぐれではないプレイヤーの安定した強さを汲み取るため|ゲーム自体の先手勝率の偏りを５割に戻すため（※注３）|
|残っている課題||TODO 千日手や、手数上限による引き分けをどう扱うかも、解決したい|

（※注１）最初、先後固定制だけでシステムを作り、その後、柿木さんから手番を交代するバージョン（先後交互制）も作るよう宿題が出たので取り掛かり、それにも対応することができました  
（※注２）例えば３本勝負というのは、過半数の２本を取れば勝ちです。（３本を取るゲームではありません）  
（※注３）［従来のｎ本勝負］と［かくきんシステム］は排他的なものではありません。単純に［かくきんシステム］の［目標の勝ち点］を２倍にすれば、（［３本勝負］とはゲーム体験の感覚は異なるものの）［３本勝負］の意義を含めることができます
